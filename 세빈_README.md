# 개인 회고 PJT6 - 프로젝트 기획

오늘은 단순히 기능을 나열하는 기획이 아니라, 서비스가 왜 존재해야 하는지부터 깊이 있게 파고드는 시간을 가졌습니다. 요즘 개발자는 단순 구현에 그치지 않고, 문제 정의 → 사용자 분석 → 해결 방안 설계까지 기획 단계에서도 적극적으로 기여해야 한다고 생각합니다. 그래서 이번 서비스 기획에도 논리적인 흐름을 갖추는 데 특히 공을 들였습니다.

먼저 페르소나와 문제 정의를 명확히 세우는 과정에서 초보 러너들이 겪는 불편과 니즈가 무엇인지 다시 한 번 점검할 수 있었습니다. “왜 이 서비스가 필요한가?”라는 질문에 스스로 납득할 수 있을 때까지 고민했고, 그 결과 서비스가 해결해야 할 핵심 포인트와 제공할 가치가 자연스럽게 도출됐습니다.

또한 해결 방안과 서비스 목표를 구체화하면서, 단순 아이디어 수준이었던 개념이 실체를 갖기 시작했습니다. 서비스가 어떤 방식으로 문제를 해결하고, 어떤 지표를 중심으로 성장해야 하는지 설정하면서 전체적인 구조가 훨씬 선명해졌습니다.

이 기반 위에서 기능 요구사항과 ERD를 작성하니 구현까지의 가시성이 크게 높아졌습니다. 어떤 데이터를 어떻게 저장하고, 어떤 기능이 어떤 흐름으로 동작해야 하는지 명확하게 보였기 때문에 이후 개발 단계에서 시행착오를 줄일 수 있을 것 같다는 확신이 들었습니다.

이제 프로토타입과 와이어프레임을 완성하면 본격적인 개발만 남았는데, 준비가 탄탄한 만큼 구현 과정이 기대되고 스스로도 더 성장할 수 있을 것 같습니다.
